# Работа БЕЗ одобрения HH.ru — полное решение

## Проблема
Регистрация приложения на dev.hh.ru требует одобрения модераторов (≈1 неделя). Пока заявка в ожидании — использовать OAuth нельзя.

---

## Решение: что доступно БЕЗ регистрации

### ✅ Работает без регистрации (0 минут)

По [официальной документации](https://habr.com/ru/companies/hh/articles/303168/):

> «Для того чтобы начать работать с данными, доступными без авторизации, вам ничего не потребуется.»

**Доступно только с заголовком `User-Agent`:**

| Функция | Эндпоинт | Статус |
|---------|----------|--------|
| Поиск вакансий | `GET /vacancies` | ✅ Работает |
| Детали вакансии | `GET /vacancies/{id}` | ✅ Работает |
| Справочники (регионы, метро, опыт) | `GET /dictionaries`, `/areas` | ✅ Работает |
| Информация о работодателе | `GET /employers/{id}` | ✅ Работает |

**Пример запроса:**
```bash
curl -H 'User-Agent: MyApp/1.0 (your@email.com)' 'https://api.hh.ru/vacancies?text=python&area=1'
```

### ❌ Требует OAuth (нужно одобрение приложения)

| Функция | Требование |
|---------|------------|
| Список резюме | OAuth |
| Отклик на вакансию | OAuth |
| История откликов | OAuth |

---

## Стратегия: гибридный режим

### Режим «Сейчас» (без HH approval)

1. **Поиск вакансий** — через публичный API (User-Agent).
2. **Детали вакансии** — полное описание, требования, зарплата.
3. **Резюме** — пользователь вставляет свой текст в приложение (или импорт из файла).
4. **Письмо** — Deep Seek генерирует сопроводительное письмо.
5. **Отклик** — вручную: кнопка «Скопировать письмо» + «Открыть вакансию на HH».

**Один лишний шаг:** скопировать письмо и вставить его в форму отклика на hh.ru.

### Режим «Позже» (после одобрения HH)

- Подключить OAuth.
- Загружать резюме с HH.
- Отправлять отклики автоматически.

## Архитектура приложения (без OAuth)

```
┌─────────────────────────────────────────────────────────────────┐
│  ВЕБ-ПРИЛОЖЕНИЕ                                                 │
│                                                                  │
│  1. Поиск вакансий ──► HH API (User-Agent) ◄── без регистрации  │
│  2. Резюме ──► пользователь вводит/вставляет текст              │
│  3. Генерация письма ──► Deep Seek API                          │
│  4. Копировать + открыть hh.ru/vacancy/XXX                      │
└─────────────────────────────────────────────────────────────────┘
```

**Что нужно:**
- HH API: только `User-Agent` (например, `JobHelper/1.0 (vladkos@yandex.ru)`)
- Deep Seek API: ключ с platform.deepseek.com
- Никаких Client ID / Client Secret от HH

---

## UX для ручного отклика

Чтобы упростить ручной отклик:

1. **Кнопка «Скопировать письмо»** — письмо в буфер обмена.
2. **Кнопка «Открыть на HH»** — новая вкладка: `https://hh.ru/vacancy/{id}`.
3. **Подсказка:** «Письмо скопировано. Откройте вакансию и вставьте его в поле сопроводительного письма (Ctrl+V)».

## Итог

| Этап | Действие HH | Что можно делать |
|------|-------------|------------------|
| Сейчас | Ничего | Поиск, детали вакансий, генерация писем, ручной отклик |
| После одобрения | Подключить OAuth | Автоотклики, загрузка резюме с HH |

Основной функционал доступен сразу, без ожидания одобрения.
