# Инструкция по установке и запуску

## Требования

- Node.js 18+ 
- npm или yarn

## Установка

1. Установите зависимости:
```bash
npm install
```

## Запуск

### Режим разработки
```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:3000

### Сборка для продакшена
```bash
npm run build
npm start
```

## Особенности

### 200-тиковый график

Для 200-тикового графика реализована следующая логика:

1. Подписка на WebSocket поток тиков (`@trade`) от Binance
2. Сбор последних 200 тиков локально в буфере
3. Конвертация тиков в свечи для отображения на графике
4. Группировка тиков по 10-секундным интервалам для визуализации
5. Обновление в реальном времени

### WebSocket подключения

- **BinanceTickStream** - для получения тиков (200-тиковый график)
- **BinanceKlineStream** - для получения обновлений свечей (стандартные таймфреймы)
- Автоматическое переподключение при обрыве связи (до 5 попыток)

### Управление состоянием

Используется Zustand для централизованного управления:
- Выбранная пара
- Таймфрейм
- Данные графика (свечи и тики)
- Состояние подключения к Binance

## API Binance

Проект использует публичные эндпоинты Binance:
- REST API (`/api/v3/klines`, `/api/v3/ticker/24hr`) для получения исторических данных
- WebSocket (`wss://stream.binance.com:9443/ws`) для обновлений в реальном времени

**Не требуется API ключ** для базовой функциональности.

## Решение проблем

### Ошибка подключения к Binance

Если возникают проблемы с подключением:
1. Проверьте интернет-соединение
2. Убедитесь, что Binance API доступен в вашем регионе
3. Проверьте консоль браузера на наличие ошибок

### График не отображается

1. Убедитесь, что выбрана валютная пара
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что WebSocket подключение установлено (индикатор статуса в хедере)

### 200-тиковый график пустой

200-тиковый график начинает заполняться только после подключения к WebSocket и получения тиков. Подождите несколько секунд после выбора пары.
